# Court Report — Mobile App Setup & Activation

The iOS and Android apps are built with Capacitor (SPM on iOS, Gradle on
Android). The web build is the source of truth; `npm run build && npx cap sync`
copies it into both native projects.

This doc lists the steps that are **blocked on your developer accounts** or must
be done in Xcode / Android Studio. Everything else is already wired in code.

---

## 0. Prerequisites (you)
- **Xcode** (Mac App Store) — for iOS.
- **Android Studio** — for Android.
- **Apple Developer Program** ($99/yr) and/or **Google Play** ($25 once).

## 1. Google sign-in redirect (required for native OAuth)
Add the deep-link URL to the Supabase allow-list or native Google login is
rejected:
> Supabase → Authentication → URL Configuration → **Redirect URLs** → add
> `com.palmvolleypickle.courtreport://auth`

## 2. App icons & splash — DONE
Generated by `@capacitor/assets` from `assets/logo.png` (navy background). To
regenerate after a logo change:
```
npx @capacitor/assets generate --iconBackgroundColor '#0a456a' \
  --iconBackgroundColorDark '#0a456a' --splashBackgroundColor '#0a456a' \
  --splashBackgroundColorDark '#0a456a'
```

## 3. iOS privacy manifest — one Xcode step
`ios/App/App/PrivacyInfo.xcprivacy` exists (declares UserDefaults + Capacitor
required-reason APIs, no tracking). **In Xcode, add it to the App target:**
select the App project → File → Add Files to "App"… → pick `PrivacyInfo.xcprivacy`
→ ensure "App" target is checked. Without target membership it's ignored and you
may get Apple's ITMS-91053 email.

## 4. Account deletion — DONE
In-app **Delete account** button (account menu) calls the
`tracker_delete_my_account` RPC — required by Apple guideline 5.1.1(v). It clears
the user's data and auth identity; shared games/groups stay for other members.

## 5. Push notifications — activation (blocked on Firebase)
Code is in place (`src/lib/push.js`, `push_tokens` table + RPCs, and the
`supabase/functions/send-push` FCM function). The plugin is synced into the
native projects, but stays **inert** (no crash) until Firebase is configured —
tokens simply never register.

> ⚠️ Before you ship a native build with push actually working, add the
> Firebase config below. If you want to build/run native BEFORE setting up
> Firebase and hit any Firebase-init issue, temporarily remove
> `@capacitor/push-notifications` (`npm rm @capacitor/push-notifications && npx
> cap sync`) — the rest of the app is unaffected.

To activate:
1. Create a **Firebase project**; add the iOS app (bundle id) and Android app
   (`com.palmvolleypickle.courtreport`).
2. **Android:** put `google-services.json` in `android/app/`, add the
   Google-services Gradle plugin, then `npx cap sync android`.
3. **iOS:** add `GoogleService-Info.plist`; upload an **APNs auth key (.p8)** to
   Firebase → Cloud Messaging; in Xcode enable the **Push Notifications**
   capability and **Background Modes → Remote notifications**.
4. **iOS token caveat:** the bare `@capacitor/push-notifications` gives a raw
   APNs token on iOS, which the FCM send function can't use. Swap in
   **`@capacitor-firebase/messaging`** so both platforms store an FCM token
   (update `src/lib/push.js` to read the FCM token). Android already yields an
   FCM token.
5. Deploy the sender:
   ```
   supabase secrets set FCM_SERVICE_ACCOUNT="$(cat service-account.json)"
   supabase secrets set SEND_PUSH_SECRET="<random>"
   supabase functions deploy send-push
   ```
   Call it with header `x-send-secret` and body
   `{ "userIds": [...], "title": "...", "body": "..." }`.

## 6. Optional Android hardening (pre-launch)
Defaults are fine to ship, but consider: `android:allowBackup="false"` in the
manifest, and `minifyEnabled true` in `android/app/build.gradle` (test with the
included `proguard-rules.pro` first).

## 7. Build & submit
```
npm run build && npx cap sync
npx cap open ios       # Xcode: set signing team, archive, upload
npx cap open android   # Android Studio: Build → Generate Signed Bundle (AAB)
```
Both stores also need the privacy policy URL:
`https://palmvolleypickle.com/court-report/privacy`.
